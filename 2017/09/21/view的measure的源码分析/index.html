view的measure过程

摘要

本文从源码的角度分析View的measure过程。然后指出了一个自定义View设置wrap_content的问题。

View的mesure过程

measure

measure是个final方法，子类无法对它扩展。真正的measure过程是在onMesure中进行的。

onMeasure

参数widthMeasureSpec和heightMeasureSpec是由parent指定的。
先看看setMeasuredDimension是干什么的。

setMeasuredDimension

setMeasuredDimension只是对mMeasuredWidth和mMeasuredHeight赋值，把measure的结果存了下来。

再看看int measuredWidth, int measuredHeight这两个参数是怎么来的。
getDefaultSize(getSuggestedMinimumWidth(), widthMeasureSpec),
getDefaultSize(getSuggestedMinimumHeight(), heightMeasureSpec)

getDefaultSize

getSuggestedMinimumHeight/Width 返回的是View应该的最小高度，即View的最小width/height和parent的最小width/hight的最大值

tips

自定义View的warp_content属性

从LayoutParams和MeasureSpec的对应关系我们可以知道，如果View的宽度/高度设置为wrap_content，那么它的MeasureSpec对应的AT_MOST。
从getDefaultSize方法可以看到，如果是AT_MOST，那么会返回parent指定的size。wrap_content属性失效。
解决方法：
重写onMeasure，在这种情况下手动设置需要的宽/高。

getMeasuredWidth、getMeasuredHeight

measure完成后，通过getMeasuredWidth、getMeasuredHeight可以获得测试宽/高，但是极端情况下会measure多次，比如源码中的那些极端的if语句中也有measure的调用。因此，要获取View的宽/高，到onLayout中获取比较保险。
