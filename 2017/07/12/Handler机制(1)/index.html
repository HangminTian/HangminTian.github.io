1.为什么需要Handler机制

  • 子线程不能访问View

  • 主线程不能被阻塞
    随便想一下，最容易想到的就是这两个原因了。如果有多个线程同时访问View，那么View的显示会错乱；而如果对访问View的操作加锁，则会降低UI性能。
    刚开始写Android代码的时候，一定碰到过这样的一个Exception：

  • postDelayed:

  • sendMessage:

  • 非主线程
    非主线程需要自己初始化Looper。
    `JAVA

    Looper.prepare();
handler = new Handler(getContext().getMainLooper()) {
    @Override
    public void handleMessage(Message msg) {
        //dosth
    }
};
Looper.loop();
